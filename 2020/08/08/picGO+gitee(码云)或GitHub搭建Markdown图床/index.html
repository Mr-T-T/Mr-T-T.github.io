PicGO+gitee（码云）/github搭建Markdown图床
發表於 2020-08-08| 更新於 2020-08-10|blog
閱讀量:

背景：

Markdown添加图片有三种方式：
![图片alt](图片地址 ‘’图片title’’)

插入本地图片：

只需要在基础语法的括号中填入图片的位置路径即可，支持绝对路径和相对路径。

缺点：不灵活不好分享，本地图片的路径更改或丢失都会造成markdown文件调不出图。在上传md文件的同时必须同时把图片文件也上传。

插入网络图片：

![图片alt](图片地址 ‘’图片title’’)
将图片存在网络服务器上，非常依赖网络。

将图片存入markdown文件

用base64转码工具把图片转成一段字符串，然后把字符串填到基础格式中链接的那个位置。

基础用法：
![avatar](data:image/png;base64,iVBORw0......)
这个时候会发现插入的这一长串字符串会把整个文章分割开，非常影响编写文章时的体验。如果能够把大段的base64字符串放在文章末尾，然后在文章中通过一个id来调用，文章就不会被分割的这么乱了。
高级用法：
比如：
![avatar][base64str]
[base64str]:data:image/png;base64,iVBORw0......

如何得到base64的图片编码呢？

使用python将图片转化为base64字符串

base64字符串转化为图片

然而我们在写博客的过程中最常用到却是截图或本地图片，基于使用本地图片的各种缺点，我们就需要一个图床了。它的作用便是一个便于在博文中插入在线图片连接的个人图片仓库。设置图床之后，在自己博客中插入的图片链接就可以随时随地在线预览了，并且不会因为任何意外原因无法查看，除非自己亲自删除。

PicGo就是为了解决这个问题诞生的，它可以将图片上传到指定的图床上，然后返回markdown链接，直接粘贴到你的文档中，就搞定啦

各种方法对比：

现在大多推荐的是七牛+ipic（ios）或mpic（Windows）搭建图床，而七牛在绑定域名域名的时候需要域名备案，但是备案是需要租赁服务器的，太麻烦还不实惠，于是放弃。而微博呢现在挂链接又很厉害。于是很多大部分人选择用github+picGo搭建图床，但是github虽好却是国外的网站，速度终究比不上国内网站，研究了小半天，终于发现完美的解决方案：

那就是使用PicGo + 国内的github - 码云来搭建markdown图床

安装picGO

首先进入官网下载并安装
picGo官网
picGo下载地址

下载对应的版本（macOS下载dmg文件，Windows下载exe文件）

下载速度真的一言难尽，还动不动下载失败！！！（解决github releases下载过慢问题可以使用第三方下载工具）

下载完毕后打开主页

点击插件设置搜索gitee下载picgo-plugin-gitee-uploader插件
（这里需注意必须先安装node.js才能下载插件，相信走到这一步的人应该都安装了吧）

创建仓库：

点击头像旁的加号创建仓库

安全令牌：也就是需要用到的token值

接下来鼠标移至头像，点击设置
找到私人令牌：

点击生成新令牌：勾选如图后提交

需要注意的是生成之后这个令牌只会明文显示一次，先复制存储起来，否则需要重新生成。

配置picGO：

点击图床设置，选择gitee
按照如图提示进行配置：

  • repo：填写 用户名/仓库名
  • branch： 分支 直接填 master
  • token：填写之前保存的 私人令牌
  • path： 任意填写 会创建相应文件夹来保存图片

配置完成后，设置为默认图床，点击确认就大功告成了。

使用：

配置完成后可能得略等一会才能正常使用

本地图片：

本地图片直接拖动到主界面上传框内即可，上传完毕后链接会自动复制到剪贴板，直接粘贴使用即可

截图：

使用截图工具截图后使用快捷键Ctrl+Shift+P直接上传，上传完毕后自动复制到剪贴板，直接粘贴使用即可
